Conceptos base
¿Qué es un Producto Digital?
Un producto digital es cualquier software que resuelve un problema real y puede distribuirse, venderse o usarse a través de internet o dispositivos.
🌐
DAW — Aplicación Web
Se ejecuta en el navegador. HTML, CSS, JavaScript, frameworks como Laravel, React, Vue. Accesible desde cualquier dispositivo con internet.
📱
DAM — Aplicación Móvil
Se ejecuta en Android o iOS. Puede ser nativa (Swift, Kotlin), híbrida (.NET MAUI, Flutter) o web progresiva (PWA).
🖥️
Aplicación de Escritorio
Se instala en Windows, Mac o Linux. WPF, Electron, .NET MAUI Desktop. Ideal para uso offline intensivo o hardware especializado.
🔗 ¿Se pueden combinar?
Sí — y es lo habitual. Una plataforma moderna tiene una app móvil (DAM) para el usuario final, una web de administración (DAW) para el gestor, y un backend API que conecta ambas. A esto se le llama arquitectura full-stack o SaaS.
📱 Ejemplo · PrecioScan
DAM: App .NET MAUI para Android/iOS — el usuario fotografía productos.
DAW: Panel Laravel para el administrador — gestiona usuarios, suscripciones, estadísticas.
API: Backend Laravel que conecta ambas y llama a servicios externos (Anthropic OCR, Google Maps).
Conceptos base
Arquitecturas de Software
La arquitectura define cómo se organiza y comunica el código. Elegir bien desde el principio ahorra meses de trabajo.
🏛️ Monolítica
Todo el código en una sola aplicación. Fácil de empezar, difícil de escalar. Válida para proyectos pequeños o MVPs.

Ventaja Simple · Riesgo Escala mal
⚙️ API REST + Clientes
Backend expone endpoints JSON. Cualquier cliente (web, móvil, escritorio) consume la misma API. Es el estándar actual.

Ventaja Flexible · Estándar Más usado
🧩 Microservicios
Cada funcionalidad es un servicio independiente. Alta complejidad, pero escala infinitamente. Para grandes empresas.

Nivel Avanzado · Riesgo Complejo
☁️ Serverless
Funciones en la nube que se ejecutan solo cuando se necesitan. Sin servidor permanente. AWS Lambda, Cloudflare Workers.

🔑 Regla de oro: las API keys van SIEMPRE en el backend
Nunca incluyas una API key en el código de la app móvil o en el JavaScript del frontend. Cualquiera puede extraerla. El backend actúa como proxy seguro: la app le pide al backend, y el backend llama a la API externa con la key secreta.

Monetización · Parte 1
De Proyecto a Negocio Digital
Saber programar no es suficiente. Un producto digital que no genera ingresos es un hobby, no un negocio. La diferencia está en la mentalidad.
🎓 Mentalidad de proyecto
Lo hago porque me lo piden
El objetivo es que funcione
El cliente es el profesor o la empresa
Se entrega y se olvida
El coste es el tiempo invertido
No hay plan de continuidad
💼 Mentalidad de producto
Lo hago porque resuelve un problema
El objetivo es que lo usen y paguen
El cliente es el usuario final
Se lanza, se mide, se mejora
El coste es una inversión con retorno
Hay un plan de crecimiento
💡 La pregunta clave antes de empezar
No es "¿puedo programar esto?" — eso ya lo sabes hacer. La pregunta es: ¿alguien pagaría por esto? ¿Cuánto? ¿Por qué? ¿Con qué frecuencia? Responder esto antes de escribir código es lo que separa a un desarrollador de un emprendedor digital.
💰
Ingresos directos
El usuario paga por usar el producto. Suscripción, pago único, pay-per-use.
📣
Ingresos indirectos
El producto es gratis, pero monetizas con publicidad, datos anonimizados o afiliación.
🏢
Ingresos B2B
Vendes el producto o el desarrollo a empresas, no a usuarios individuales.

⚖️ La regla del 30% — el gran debate
Google y Apple cobran un 30% de cada compra dentro de la app. Para suscripciones anuales baja al 15% a partir del segundo año. Por eso muchas apps redirigen al usuario a pagar en su web propia (sin comisión) y luego acceder desde la app. Esto es legal pero con restricciones — no puedes poner el enlace de pago dentro de la app en iOS.

Monetización · Parte 6
Hoja de Ruta: de 0€ a Ingresos Recurrentes
Un plan realista paso a paso. No hay atajos, pero sí una secuencia que minimiza el riesgo y maximiza el aprendizaje.
📋 Paso 1 — Valida la idea (semana 1-2)
Habla con 10 posibles usuarios. No preguntes si usarían la app — pregunta cómo resuelven hoy el problema.
Crea una landing page en Carrd.co o Framer (gratis) con botón "Únete a la lista de espera".
Si consigues 50 emails: la idea tiene tracción. Si no: pivota.
🔨 Paso 2 — Construye el MVP (mes 1-3)
Solo las funcionalidades imprescindibles. En PrecioScan: foto + GPS + precio + listado.
Sin backend, sin nube, sin OCR. Eso viene después.
Regla: si no duele cortar una feature, córtala.
🚀 Paso 3 — Lanza y mide (mes 3-4)
Publica en Google Play (APK o Store). Lanza en ProductHunt, Reddit y Twitter.
Instala Firebase Analytics (gratis) para saber cómo usan la app.
Métrica clave: ¿Cuántos vuelven al día siguiente?
💳 Paso 4 — Activa pagos (mes 4-6)
Añade el modelo freemium con Google Play Billing o Stripe (web).
Precio inicial conservador: 0,99€ - 1,99€/mes.
Con 100 suscriptores a 1,99€ = 199€/mes recurrentes.
No es mucho, pero es la prueba de que alguien paga.
📈 Paso 5 — Crece (mes 6+)
Añade las features que piden los usuarios que ya pagan.
Lanza en App Store (iOS) para doblar el mercado.
Busca el canal de adquisición que mejor convierte:
SEO, anuncios, partnerships, afiliados.
500 suscriptores = 995€/mes. 2.000 = 3.980€/mes.
🏢 Paso 6 — Versión B2B (año 2)
Ofrece la app a empresas con funciones premium: multi-usuario, panel de gestión, API, integración con sus sistemas.
Precio B2B: 50-500€/mes por empresa.
10 empresas a 100€/mes = 1.000€/mes extra.
Monetización · Parte 7
Los 8 Errores que Hunden un Producto Digital
Lo que nadie te cuenta en clase. Estos errores no son de código — son de estrategia. Y son los más caros.
❌ Error 1 — Construir sin validar
6 meses programando algo que nadie necesita. Solución: habla con usuarios antes de escribir código.
❌ Error 2 — Esperar a que esté "perfecto"
"Lo lanzo cuando esté listo." Resultado: nunca se lanza. Un MVP imperfecto publicado vale más que una obra maestra en el cajón.
❌ Error 3 — No cobrar desde el principio
"Ahora es gratis, luego cobraré." Los usuarios gratuitos raramente convierten. Es mucho más difícil cobrar a alguien que ya usaba gratis.
❌ Error 4 — Ignorar la retención
Obsesionarse con descargas en lugar de con usuarios que vuelven. 10.000 descargas sin retención = 0 negocio.
⚠️ Error 5 — Competir solo por precio
Bajar el precio para conseguir usuarios destruye el margen. Compite por valor, no por precio.
⚠️ Error 6 — No tener un canal de adquisición
"La app es tan buena que se vende sola." No existe. Necesitas un canal: SEO, anuncios, comunidad, partnerships o ventas directas.
⚠️ Error 7 — Olvidar los aspectos legales
Política de privacidad, RGPD, términos de uso, IVA digital europeo. Son obligatorios y evitan multas graves.
💡 Error 8 — Trabajar solo sin red
Comunidades como IndieHackers, MicroConf o grupos de Telegram/Discord de desarrolladores indie son fuente de feedback, motivación y primeros clientes.

Fiscalidad · Parte 1
Tributación como Autónomo en España
Si desarrollas y vendes productos digitales como persona física, eres autónomo. Conocer tus obligaciones fiscales desde el principio evita sustos con Hacienda.
📋 Alta y obligaciones
Alta RETA Seguridad Social de autónomos. Cuota mínima: ~230€/mes (2024, base mínima).

Alta censal Modelo 036/037 en Hacienda. Epígrafe IAE: 763 (programación informática) o 762 (consultoría).

Tarifa plana 80€/mes los primeros 12 meses si es tu primer alta como autónomo.
💸 IVA — Impuesto sobre el Valor Añadido
Tipo general: 21% en servicios digitales en España.

Modelo 303 Declaración trimestral (abril, julio, octubre, enero).

Modelo 390 Resumen anual de IVA (enero del año siguiente).

IVA intracomunitario: si vendes a otros países de la UE, usa el sistema OSS (One Stop Shop) para no registrarte en cada país.
💶 IRPF — Impuesto sobre la Renta
Modelo 130 Pago fraccionado trimestral: 20% del beneficio neto cada trimestre.

Modelo 100 Declaración anual de la Renta (mayo-junio).

Tramos IRPF 2024 (beneficio neto):
· Hasta 12.450€ → 19%
· 12.450 - 20.200€ → 24%
· 20.200 - 35.200€ → 30%
· 35.200 - 60.000€ → 37%
· Más de 60.000€ → 45-47%
📁 Gastos deducibles
✅ Equipos informáticos (amortización)
✅ Software, licencias, suscripciones (VS, GitHub...)
✅ Hosting, dominios, APIs de pago
✅ Formación relacionada con la actividad
✅ Parte proporcional del ADSL/móvil
✅ Gestoría y asesoría fiscal
✅ Publicidad y marketing digital
⚠️ Guardar todas las facturas — sin factura no hay deducción
Fiscalidad · Parte 2
Tributación como Empresa (SL) en España y Europa
Cuando los ingresos superan ~40.000€/año o quieres socios, una Sociedad Limitada (SL) ofrece ventajas fiscales y protege tu patrimonio personal.
🏢 Constitución de una SL
Capital mínimo: 1€ (reforma 2023, antes 3.000€).
Coste de constitución: ~500-1.000€ (notario + registro).
Tiempo: 5-15 días hábiles.

Alternativa rápida SL Express: 24-48h a través del CIRCE (Centro de Información y Red de Creación de Empresas) de forma telemática y gratuita.
💶 Impuesto de Sociedades (IS)
Modelo 200 Declaración anual (6 meses tras cierre del ejercicio).

Modelo 202 Pagos fraccionados (abril, octubre, diciembre).

Tipos:
· 23% tipo general
· 15% primeros 2 años para empresas nuevas
· 15% empresas con <1M€ de facturación

Ventaja clave Puedes reinvertir beneficios en la empresa antes de tributar.
🌍 IVA digital europeo — OSS
Si vendes apps o SaaS a consumidores de otros países de la UE, debes aplicar el IVA del país del comprador.

OSS (One Stop Shop) Te registras una sola vez en la AEAT española y declaras todo el IVA europeo en un único modelo trimestral (Modelo 369).

Umbrales: si ventas intraUE > 10.000€/año → obligatorio registrarse en OSS.

Legal
Cómo Hacer que tu Producto Digital sea Legal
La legalidad no es opcional — una app que maneja datos de usuarios está sujeta a leyes europeas con multas de hasta el 4% de la facturación anual.
🔒 RGPD — Reglamento General de Protección de Datos
Obligatorio en toda la UE desde 2018.

Lo que debes tener:
✅ Política de privacidad clara y accesible
✅ Consentimiento explícito para recoger datos
✅ Derecho al olvido (borrar datos del usuario)
✅ Registro de actividades de tratamiento
✅ Aviso de cookies si tienes web

Multa máxima 20 millones € o 4% facturación global
📄 Documentos legales obligatorios
Política de Privacidad — qué datos recoges, para qué, quién los trata, cuánto tiempo se guardan.

Términos y Condiciones — reglas de uso, limitaciones de responsabilidad, ley aplicable.

Política de Cookies — si tienes web. Obligatoria por la LSSI.

Aviso Legal — datos del responsable (nombre, CIF, dirección, email).
⚖️ Licencias de software
MIT Libre uso, modificación y distribución. Solo exige mantener el copyright.

Apache 2.0 Como MIT pero con protección de patentes.

GPL Si usas código GPL, tu app también debe ser GPL (open source obligatorio).

Propietaria Código cerrado, todos los derechos reservados.

⚠️ Revisa las licencias de cada NuGet/npm que uses. Algunas prohíben uso comercial.
🏷️ Propiedad intelectual
El código que creas es tuyo por defecto (derecho de autor en España).

Si trabajas para una empresa: el código pertenece a la empresa salvo pacto contrario.

Registrar la marca (nombre + logo) en la OEPM: ~150€. Recomendable antes de lanzar públicamente.

NDA (acuerdo de confidencialidad): firma siempre uno antes de mostrar tu idea a terceros.

Calidad
Normas de Calidad en Desarrollo de Software
La calidad no es hacer que funcione — es hacer que funcione bien, de forma predecible, mantenible y segura. Hay estándares internacionales para ello.
📐 ISO/IEC 25010 — Calidad del producto software
El estándar más importante para evaluar la calidad de una aplicación. Define 8 características:

Funcionalidad ¿Hace lo que debe?
Rendimiento ¿Es rápido y eficiente?
Compatibilidad ¿Funciona con otros sistemas?
Usabilidad ¿Es fácil de usar?
Fiabilidad ¿Falla poco? ¿Se recupera?
Seguridad ¿Protege los datos?
Mantenibilidad ¿Es fácil de modificar?
Portabilidad ¿Funciona en distintos entornos?
🏭 CMMI — Madurez del proceso
Mide la madurez de los procesos de desarrollo de una organización. 5 niveles:

1️⃣ Inicial — caótico, depende de personas
2️⃣ Gestionado — procesos básicos definidos
3️⃣ Definido — procesos estándar documentados
4️⃣ Cuantitativamente gestionado — métricas
5️⃣ En optimización — mejora continua

La mayoría de startups opera en nivel 1-2. El objetivo es llegar a nivel 3.
🧪 Tipos de testing obligatorios
Unit testing Prueba funciones individuales. xUnit (.NET), PHPUnit (Laravel).

Integration testing Prueba que los módulos funcionan juntos. APIs, BBDD.

UI testing Prueba la interfaz de usuario. Appium (móvil), Playwright (web).

Security testing OWASP ZAP, pruebas de inyección SQL, XSS.

Performance testing JMeter, k6 — ¿aguanta carga?

Regla mínima: cobertura de tests >70% en código crítico.
📋 ISO 9001 y buenas prácticas
ISO 9001 Sistema de Gestión de Calidad. Aplicable a cualquier empresa, incluso software.

Buenas prácticas mínimas:
✅ Código revisado por otra persona (code review)
✅ Documentación actualizada con cada versión
✅ Control de versiones (Git) desde el día 1
✅ Entorno de staging antes de producción
✅ Rollback plan: cómo volver atrás si algo falla
✅ README claro en cada repositorio
